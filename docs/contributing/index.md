# Build with SkillForge

SkillForge keeps the contribution path intentionally simple:

1. Start from a real skill folder in `skills/<slug>/`.
2. Keep the change focused and reviewable.
3. Run the repo checks before opening a PR.
4. Make sure docs and install surfaces stay honest.

# Roadmap

<p class="omcp-lead">
  What SkillForge is building, what it is considering, and what it is deliberately not doing.
  This roadmap is public because the project is public.
</p>

## Now

Active work or next in queue.

- Richer marketplace filtering — search, multi-domain filters, client compatibility badges
- Downloadable category bundles from CI artifacts
- More built-in validation hooks for pack authors
- Deeper dashboard replay workflows for debugging failed runs

## Next

Planned but not yet started.

- Benchmark leaderboards for skill-model fit
- Community contribution standards for marketplace entries
- Improved search and browsing performance on the docs site
- Pack dependency declarations (one pack can reference another)

## Later

On the horizon. Timing depends on sponsorship and contributor momentum.

- Signed marketplace releases with provenance
- Community skill scoring and quality signals
- Optional remote registry synchronization
- Pack versioning with semantic version constraints

## Deliberately not building

Some things are out of scope by design:

- **Shell execution** — SkillForge does not run commands on your machine
- **Proprietary marketplace** — the catalog stays free, public, and forkable
- **Gamification** — no badges, points, or leaderboards that the maintainer cannot keep accurate
- **Multi-tenant SaaS** — this is an open-source tool, not a hosted service
